📡 Section B: Core Queries & Join Questions

1
"Show me all registered users along with their designated roles."
Basic Projection & Selection
Queries the core authenticating user records to inspect users and roles. 
SELECT id, name, email, role 
FROM users;
Sample Command Output
id name email role
1 Administrator administrator@hostel.com admin
2 Warden warden@hostel.com warden
3 John Doe student@hostel.com student
2
"Can you join tables to show student profiles with their accounts?"
Inner Join (2 Tables)
Joins the users and students tables to retrieve profile records. 
SELECT s.student_id, u.name, u.email, s.usn, s.course, s.branch 
FROM students s
JOIN users u ON s.user_id = u.id;
Sample Command Output
student_id name email usn course branch
1 John Doe student@hostel.com 1RV22CS045 B.Tech CSE
3
"How do you inspect which rooms are currently available for allocation?"
Filtering & ENUM evaluation
Selects all available rooms. 
SELECT room_number, block, floor, capacity, occupied, status 
FROM rooms 
WHERE status = 'available';
Sample Command Output
room_number block floor capacity occupied status
101 A 1 3 1 available
4
"Show me the name and USN of students residing in Room 101."
Triple Table Join
Joins **users**, **students**, and **rooms** to locate resident listings. 
SELECT u.name, s.usn, s.course, s.branch, r.room_number
FROM students s
JOIN users u ON s.user_id = u.id
JOIN rooms r ON s.room_id = r.room_id
WHERE r.room_number = '101';
Sample Command Output
name usn course branch room_number
John Doe 1RV22CS045 B.Tech CSE 101
5
"List all students who have pending or partial fee payments."
Financial Audit & Range Selection
Queries the **fees** ledger to identify outstanding invoices. 
SELECT f.fee_id, u.name, f.amount, f.due_date, f.status 
FROM fees f
JOIN students s ON f.student_id = s.student_id
JOIN users u ON s.user_id = u.id
WHERE f.status IN ('unpaid', 'partial');
Sample Command Output
fee_id name amount due_date status
1 John Doe 67000.00 2026-06-01 unpaid

📊 Section C: Advanced Aggregates & Dashboard Statistics

6
"What is the total number of students registered in the hostel?"
COUNT(*) Cardinality Aggregation
SELECT COUNT(*) as total_students FROM students;
7
"Show me the count of students Present, Absent, and On Leave today."
Group By Frequency
SELECT status, COUNT(*) as count 
FROM student_attendance 
WHERE date = CURDATE() 
GROUP BY status;
8
"How many unique students have filed at least one complaint?"
COUNT(DISTINCT) Field Evaluation
SELECT COUNT(DISTINCT student_id) as unique_students_complained 
FROM complaints;
9
"Give me the breakdown of hostel complaints grouped by their status."
GROUP BY Status Frequency
SELECT status, COUNT(*) as complaint_count 
FROM complaints 
GROUP BY status;
10
"How many times have notice board bulletins been published?"
Standard Count
SELECT COUNT(*) as total_notices_published FROM notices;
11
"What is the total sum of outstanding (unpaid) fees?"
SUM() Aggregation Function
SELECT SUM(amount) as total_unpaid_fees 
FROM fees 
WHERE status = 'unpaid';

🧠 Section D: The Examiner's Advanced Audit Queries

12
"Prove that your database has absolute Referential Integrity and contains no orphaned student records (i.e. students without a user)."
Outer Join (LEFT JOIN) filtering for NULL pointers
A LEFT JOIN retrieves all students. If any student has NO corresponding account in the users table, users.id will be NULL. The result should be 0 rows, proving 100% referential integrity! 
SELECT s.student_id, s.usn 
FROM students s 
LEFT JOIN users u ON s.user_id = u.id 
WHERE u.id IS NULL;
13
"Write a Nested Subquery to find students with fees higher than the average fee."
Non-Correlated Subquery & Selection Comparison
The nested inner query computes the average fee amount. The outer query dynamically filters students whose outstanding invoice exceeds this benchmark. 
SELECT u.name, s.usn, f.amount
FROM students s
JOIN users u ON s.user_id = u.id
JOIN fees f ON s.student_id = f.student_id
WHERE f.amount > (SELECT AVG(amount) FROM fees);
14
"Audit private messages showing both sender and recipient names."
Double Join on the Same Table (Aliases)
Both sender and recipient IDs link back to users.id. Joining the users table twice under unique aliases allows showing sender and recipient names side-by-side. 
SELECT m.message_id, sender.name as sender_name, recipient.name as recipient_name, m.content 
FROM private_messages m
JOIN users sender ON m.sender_id = sender.id
JOIN users recipient ON m.recipient_id = recipient.id
ORDER BY m.created_at DESC;
15
"Show me which rooms currently have bed vacancies."
Attribute Comparison Logic
Compares the current occupied counter against capacity, calculating the remaining open slots. 
SELECT room_number, block, floor, capacity, occupied, (capacity - occupied) as vacant_beds 
FROM rooms 
WHERE occupied < capacity;
16
"What is the average outstanding fee balance grouped by study branch?"
Joins Coupled with GROUP BY Aggregation
Groups outstanding fee records by study branches to display demographic summaries. 
SELECT s.branch, COUNT(s.student_id) as student_count, ROUND(AVG(f.amount), 2) as avg_outstanding_fee
FROM students s
JOIN fees f ON s.student_id = f.student_id
WHERE f.status = 'unpaid'
GROUP BY s.branch;
17
"Provide a complete attendance audit log showing the student, date, status, and the staff member who marked it."
Double Join to Audit Accountability
Joins users twice: once for the student name and once for the staff name, tracking marking records. 
SELECT sa.date, u.name as student_name, sa.status, staff.name as marked_by_staff, staff.role as staff_role
FROM student_attendance sa
JOIN students s ON sa.student_id = s.student_id
JOIN users u ON s.user_id = u.id
JOIN users staff ON sa.marked_by = staff.id
ORDER BY sa.date DESC;
